OVERVIEW
--------
Ketoan ERP is a multi-company accounting system (Vietnamese, TT200). It contains a Node.js/Express backend using PostgreSQL and a React + Vite frontend with Tailwind CSS.

KEY FILES & PURPOSE
-------------------
- backend/server.js
  - Initializes DB tables (companies, users, user_companies, sessions, etc.)
  - Provides auth endpoints: /api/auth/register-admin, /api/auth/login, /api/auth/logout
  - User management: /api/users, POST /api/users, DELETE /api/users/:id
  - Company management: /api/companies
  - Uses JWT + sessions table to validate tokens via `authenticate` middleware

- front-end/src/context/AuthContext.jsx
  - Holds `token`, `user`, `companies`, `users` state
  - Exposes `loadUsers()` used by components instead of calling API directly
  - Manages `activeCompany` in localStorage and `token` in sessionStorage

- front-end/src/views/admin/CompanyManagement.jsx
  - Admin UI for creating users, assigning companies, resetting passwords
  - Now uses `loadUsers()` from `AuthContext` to derive `localUsers`

- front-end/src/components/MainContent.jsx
  - Central place that loads data for tabs including the users management tab
  - Updated to use `loadUsers()` instead of direct API calls

RECENT CHANGES

- Replaced direct `/api/users` calls in multiple components with `AuthContext.loadUsers()` to keep a single source of truth for users and avoid race conditions where hiding admin could appear to clear user lists.

ENVIRONMENT & REQUIRED VARIABLES

  - PORT (optional, default 5000)
- front-end/.env (optional)
  - VITE_API_URL=http://localhost:5000

LOCAL SETUP & RUN (QUICK)
-------------------------
1) Backend
   - cd backend
   - npm install
   - Create `.env` from `.env.example`, set `DATABASE_URL` and `JWT_SECRET`.
   - Start server (dev): `node server.js` or `npm run dev` (requires nodemon globally)
   - Health check: GET http://localhost:5000/api/health

2) Frontend
   - cd front-end
   - npm install
   - Create `.env` (optional) with `VITE_API_URL=http://localhost:5000`
   - Start dev server: `npm run dev` (Vite default port 3000)
   - Open http://localhost:3000

TESTING THE USER LIST (ADMIN FLOW)
----------------------------------
1. Create admin (only first-run): POST /api/auth/register-admin {username, password}
2. Login: POST /api/auth/login -> receive token and user payload (saved to sessionStorage by UI)
3. The UI (AuthContext) calls `loadUsers()` during login; components read from context and render lists.
4. Verify Admin views:
   - Open Admin → "Cấu hình hệ thống pháp nhân" or tab 'Quản lý ô tích phân quyền'
   - Confirm user list shows admin, KTT, NV as applicable

NOTES & TROUBLESHOOTING
-----------------------
- If users are not visible: verify backend `/api/users` returns rows when called with Authorization header (Bearer token). If direct API returns empty, then investigate DB records or the SQL query filtering by role or manager.
- Session handling: tokens are validated against `sessions` table; ensure the session row exists after login.
- If using `npm run dev` for backend, `nodemon` must be installed globally or as a dev dependency.
